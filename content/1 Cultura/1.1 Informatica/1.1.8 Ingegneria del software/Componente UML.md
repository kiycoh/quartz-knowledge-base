# Componente UML
Un componente, secondo la definizione UML, è una parte fisica e sostituibile di un sistema che ne incapsula l'implementazione. Può rappresentare, ad esempio, file di codice sorgente, librerie, file eseguibili o altri elementi che compongono il software.
